# Learning: 2026-04-15

## Context

Full eval run across 20 forked public repos (laura-mlg) spanning Python, TypeScript, Ruby, Go, Java, Kotlin, PHP, Rust, Swift, Shell, Docker, Terraform, SQL, OpenAPI, and monorepo scenarios. 4 agents ran full autonomous sessions (shoelace, timber, clap, spring-petclinic); 1 was manually reviewed (cobra); 15 were analyzed directly and committed via GitHub API.

## What we learned

1. **Wrong field name (`config =` vs `config_files = [...]`)**: The cobra agent used `config = ".golangci.yml"` — not a valid TOML field. The skill mentioned `config_files` in one example but lacked a universal rule. **Fix:** Added explicit rule to Decision 3: "Always use `config_files = ["path"]`. Never use `config = "path"`."

2. **golangci-lint + gofmt double-up**: The cobra run enabled both, even though the skill said to prefer golangci-lint. The "prefer" wording was too soft. **Fix:** Changed to "if `.golangci.*` exists, use ONLY `golangci-lint` — do NOT also add `gofmt`."

3. **Invalid trivy driver `fs-vuln`**: cobra used `drivers = ["config", "fs-vuln"]`. Valid values are `"config"`, `"secret"`, and `"vuln"` only. **Fix:** Enumerated all valid driver values and added explicit "Note: `fs-vuln` is NOT a valid driver value."

4. **Security scanners defaulted to `monitor` in modes table**: User confirmed security scanners should be `block`. The original table had `osv-scanner`, `trivy`, `checkov`, `semgrep`, `bandit` all as `monitor`. **Fix:** Split the row — security/vulnerability scanners are now `block`; `tflint` moved to `comment`; remaining noisy tools stay `monitor`. Also updated the Phase 4 summary example accordingly.

5. **Biome exclusivity was a soft note, not a hard rule**: The zod repo uses `biome.jsonc`, not eslint. The skill had a parenthetical note but agents could ignore it. **Fix:** Made it a bold hard rule in both the Phase 1 config table and the Decision 1 selection rules.

6. **No autonomous fallback path documented**: Agents running without a user had to infer defaults from the interactive prompts. **Fix:** Added explicit note at the top of Phase 3: "If running autonomously, apply all recommended defaults and proceed without confirmation."

7. **`extra_packages` version resolution gap**: Skill said to map plugins to npm packages but didn't clarify whether to use devDependency versions or latest. **Fix:** Added rule: "Use versions already pinned in `package.json` devDependencies wherever possible."

8. **`[[source]]` block unexplained**: Agents questioned whether it was required. **Fix:** Added comment `# Required — points to the built-in Qlty plugin registry.` in the Phase 4 template.

9. **`tsc` missing from Decision 2 modes table**: tsc was in Decision 1 selection rules but not in the modes table. (Note: already present in the table at time of this diff; confirmed no action needed.)

10. **Stylelint recommended for any CSS file**: Some repos have CSS files without using stylelint at all. **Fix:** Added condition — only recommend stylelint if CSS/SCSS files exist AND a stylelint config is found OR stylelint is in devDependencies.

11. **zizmor not listed on docs.qlty.sh/plugins**: Agents questioned or omitted it. **Fix:** Added note to the Phase 1 config table: "zizmor may not appear on docs.qlty.sh/plugins — it IS a supported Qlty plugin."

12. **Monorepo prefix scoping unclear for same-language workspaces**: The clap agent wasn't sure whether to scope a Cargo workspace. **Fix:** Added explicit guidance to Decision 7: "For same-language workspaces (e.g., all Rust crates), do NOT scope by prefix — a single config covers all."

13. **Overly broad exclude patterns matched source code**: cobra excluded `**/templates/**` and `**/config/**`, which matched actual source files in a config/template library. **Fix:** Added warning to Decision 4: "Avoid broad excludes like `**/config/**` or `**/templates/**` — these names are commonly used for actual source code in framework/library repos."

14. **yamllint omitted when only GitHub Actions YAML was present**: Several agents skipped yamllint because they associated it only with standalone YAML config files. **Fix:** Clarified the trigger: "always add yamllint alongside actionlint when `.github/workflows/` exists."

15. **zizmor trigger missed when `zizmor.yml` already in CI**: The `psf/requests` repo has `.github/workflows/zizmor.yml` — a strong signal. **Fix:** Added: "Strong signal: include zizmor if `.github/workflows/zizmor.yml` already exists (team is already using it in CI)."
